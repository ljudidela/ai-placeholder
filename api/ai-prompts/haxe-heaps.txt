ЖЁСТКИЕ ПРАВИЛА ДЛЯ НОВЫХ HEAPS-ПРОЕКТОВ:

1. Главный класс — src/Main.hx, наследуется от hxd.App
2. Обязательные файлы в корне:
   • project.xml          ← вместо устаревшего haxelib.json
   • build.hxml           ← основной билд-файл
   • .gitignore           ← с bin/, export/ и т.д.
3. Ресурсы (картинки, звуки, шрифты) — строго в папке res/
   → Heaps автоматически подхватит их через hxd.Res
4. Зависимости в project.xml:
   <haxelib name="heaps"/>
   <haxelib name="hlsdl"/>        ← для SDL2 (окна, ввод)
   <haxelib name="hlhashlink"/>   ← если используешь HashLink (рекомендуется)
5. build.hxml — минимальный рабочий:
   -main Main
   --hl bin/main.hl
   -lib heaps
   -lib hlsdl
   -lib hlhashlink
   -D resourcesPath=res
   --dce full
6. Запуск:
   haxe build.hxml
   hl bin/main.hl
7. README.md — обязателен, с точными командами:
   haxelib install heaps hlsdl hlhashlink
   haxe build.hxml
   hl bin/main.hl
8. Целевая платформа по умолчанию — HashLink (hl) — самый быстрый и удобный в 2025
9. НИКОГДА не используй HaxeFlixel, если явно не запрошено
10. Пакеты соответствуют структуре папок:
    src/ui/Menu.hx → package ui;
    src/entities/Player.hx → package entities;
